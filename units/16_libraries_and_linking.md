# Unit 16: Libraries & Dynamic Linking

## Overview

Every C program you have written so far uses libraries — `libc` for `printf` and `malloc`, `libpthread` for threads. This unit explains how libraries work: how to create your own static and shared libraries, how the dynamic linker resolves symbols at runtime, and how `dlopen` enables plugin systems. Understanding linking demystifies linker errors, library version conflicts, and the `LD_PRELOAD` trick used for profiling and sandboxing.

## Concepts

### Static vs Dynamic Linking

**Static linking** (`-static`): the library code is copied into your executable at link time. The resulting binary is self-contained — no dependencies at runtime. Larger binary, no version conflicts.

**Dynamic linking** (default): the executable contains references to shared libraries (`.so` files). The dynamic linker resolves them at program startup. Smaller binary, libraries can be updated without relinking, but dependencies must be present on the target system.

```bash
# Dynamic (default)
gcc program.c -o program_dynamic

# Static
gcc -static program.c -o program_static

ls -lh program_dynamic program_static
# Static binary is typically 10-100x larger
```

### Creating a Static Library

A static library is an archive of object files:

```bash
# Compile object files
gcc -Wall -c math_utils.c -o math_utils.o
gcc -Wall -c string_utils.c -o string_utils.o

# Create the archive (library)
ar rcs libmyutils.a math_utils.o string_utils.o

# Link against it
gcc main.c -L. -lmyutils -o program
# -L. = look for libraries in current directory
# -lmyutils = link libmyutils.a
```

The `ar` flags: `r` = insert/replace, `c` = create archive, `s` = write symbol table.

### Creating a Shared Library

```bash
# Compile position-independent code (required for shared libs)
gcc -Wall -fPIC -c math_utils.c -o math_utils.o

# Link into a shared library
gcc -shared -Wl,-soname,libmyutils.so.1 \
    math_utils.o -o libmyutils.so.1.0

# Create symlinks (conventional structure)
ln -s libmyutils.so.1.0 libmyutils.so.1
ln -s libmyutils.so.1   libmyutils.so

# Compile and link program against it
gcc main.c -L. -lmyutils -o program
```

The soname (`-soname,libmyutils.so.1`) is the library's canonical name embedded in the `.so` — it is the name the dynamic linker looks for at runtime, allowing the library to be updated (e.g. `libmyutils.so.1.1`) without changing the soname.

### Finding Shared Libraries at Runtime

The dynamic linker (`ld.so`) searches for shared libraries in this order:

1. `DT_RPATH` embedded in the binary (deprecated)
2. `LD_LIBRARY_PATH` environment variable
3. `/etc/ld.so.cache` (generated by `ldconfig`)
4. `/lib`, `/usr/lib`

```bash
# Temporary: set search path for this session
export LD_LIBRARY_PATH=/path/to/mylibs:$LD_LIBRARY_PATH
./program

# Permanent: embed the path in the binary at link time
gcc main.c -L/path/to/mylibs -lmyutils \
    -Wl,-rpath,/path/to/mylibs -o program

# System-wide: install the library and update the cache
sudo cp libmyutils.so.1.0 /usr/local/lib/
sudo ldconfig
```

### Inspecting Binaries and Libraries

```bash
ldd ./program               # list shared library dependencies
nm ./program                # list symbols (U=undefined, T=defined in text)
nm -D libmyutils.so         # dynamic symbols of a shared lib
objdump -d ./program        # disassemble
objdump -x ./program        # all headers
readelf -h ./program        # ELF header
readelf -d ./program        # dynamic section (libraries, rpath)
strings ./program           # print embedded strings
```

Useful patterns:
```bash
ldd ./program | grep "not found"   # find missing libraries
nm ./program | grep " U "           # find undefined (unresolved) symbols
```

### dlopen — Runtime Loading

Load a shared library at runtime, without linking against it at build time:

```c
#include <dlfcn.h>

// Load the library
void *handle = dlopen("./libplugin.so", RTLD_LAZY);
if (!handle) {
    fprintf(stderr, "%s\n", dlerror());
    exit(1);
}

// Clear any previous error
dlerror();

// Look up a symbol
typedef int (*plugin_func_t)(int, int);
plugin_func_t func = dlsym(handle, "plugin_add");

char *error = dlerror();
if (error) {
    fprintf(stderr, "%s\n", error);
    exit(1);
}

// Call the function
int result = func(3, 4);
printf("plugin_add(3, 4) = %d\n", result);

// Unload
dlclose(handle);
```

Link with `-ldl`:
```bash
gcc program.c -o program -ldl
```

RTLD flags:
| Flag | Meaning |
|------|---------|
| `RTLD_LAZY` | Resolve symbols when first used |
| `RTLD_NOW` | Resolve all symbols immediately (fail early) |
| `RTLD_GLOBAL` | Make symbols available to subsequent dlopen calls |
| `RTLD_LOCAL` | Default — symbols not visible to others |

### Symbol Visibility

By default, all symbols in a shared library are exported. Hiding internal symbols improves load time and prevents API confusion:

```c
// Mark as public API
__attribute__((visibility("default")))
int my_public_function(int x);

// Mark as internal (hidden from outside the .so)
__attribute__((visibility("hidden")))
int my_internal_helper(int x);
```

Or compile with `-fvisibility=hidden` to hide everything by default, then explicitly mark public symbols with `visibility("default")`.

### LD_PRELOAD

Load a shared library before all others — overrides symbols from libc or other libraries:

```bash
# Replace malloc with a custom allocator
LD_PRELOAD=/path/to/mymalloc.so ./program

# Use with existing tools:
LD_PRELOAD=/usr/lib/x86_64-linux-gnu/libfakeroot/libfakeroot-tcp.so command
```

This is how many profilers, debuggers, and sandboxes work.

### pkg-config

System libraries publish their compiler and linker flags via `pkg-config`:

```bash
pkg-config --cflags libssl        # get compiler flags
pkg-config --libs libssl          # get linker flags
pkg-config --modversion libssl    # get version

# Use in Makefile:
CFLAGS  += $(shell pkg-config --cflags openssl)
LDFLAGS += $(shell pkg-config --libs openssl)
```

In CMake:
```cmake
find_package(OpenSSL REQUIRED)
target_link_libraries(myprogram OpenSSL::SSL OpenSSL::Crypto)
```

### Exercise 5: Wrap malloc for Leak Detection

Write a shared library `trackalloc.so` that wraps `malloc` and `free` using `LD_PRELOAD`:

```c
// Get the real malloc/free via RTLD_NEXT
void *malloc(size_t size) {
    static void *(*real_malloc)(size_t) = NULL;
    if (!real_malloc) real_malloc = dlsym(RTLD_NEXT, "malloc");
    void *ptr = real_malloc(size);
    // record allocation
    return ptr;
}
```

Track all allocations and report unfreed memory at program exit using `atexit`. Test against a known-leaky program.
